# 运行说明

## 环境要求

- Python: `D:/software/anaconda3/python.exe`（3.12.13；若不可用可改用本机等价 3.12 解释器）
- 依赖安装：

```powershell
python -m pip install -r requirements.txt
```

## 运行测试

```powershell
python -m pytest -q
```

## 冒烟运行

```powershell
python main.py --seed 42
```

运行后会在 `output/` 目录生成：

- `latest_schedule.json`：包含 `scheduled_items`、`unscheduled_tasks`、`objective_value`、`constraint_stats`
- `cycle_log.jsonl`：每行一个周期日志，含周期状态快照、已选任务、未规划任务、约束违规统计与目标值
- `latest_task_pool.json`：任务池快照，包含 `schema_version`、`task_count`、`tasks`
- `latest_visibility_windows.json`：可见窗口池快照，包含 `schema_version`、`seed`、`horizon`、`window_count`、`windows`

## 配置说明

配置目录：`config/`（按类型拆分）

- `runtime.json`：时间轴、求解超时、随机种子
- `simulation.json`：任务数量、DAG 组数、窗口与依赖生成参数、关键任务参数
- `constraints.json`：资源容量（含并发核心阈值）、姿态角转动时间系数、热电限制等
- `objective_weights.json`：收益与惩罚权重
- `replan.json`：重规划触发阈值与等级规则
- `logging.json`：输出目录与文件名

### simulation.json（M3/M4 收敛后）

- `structured_task_ratio`：结构化任务（有 payload/窗口约束）占比，范围 `[0, 1]`。
	- 默认值：`0.82`（对应自由任务约 `0.18`）。
	- 下限保护：若配置小于 `0.6`，加载与生成阶段会按 `0.6` 执行（并在加载时给出 warning）。
	- `flex`（自由任务）价值分布在 M4 中做了轻微收敛，用于减少大量低价值自由任务对求解空间的挤占。
- `dependency_density`：结构化任务的附加依赖密度，范围 `[0, 1]`，内部映射到前驱注入概率。
- `window_reuse_target`：单窗口目标复用任务数（正数），内部换算为窗口复用下限/上限。
- 保留：
	- `visibility_window_count_min/max`
	- `visibility_window_duration_min/max`
	- `key_task_probability`
	- `key_task_value_bonus`
	- `max_predecessors`

### simulation.json 参数清理

- 已移除且当前不生效的旧参数：
	- `duration_min/duration_max`
	- `value_min/value_max`
	- `attitude_angle_min/attitude_angle_max`
- 若仍在本地配置中保留这些键，不会影响当前 M4 生成行为（兼容 M3 配置），建议删除以避免误解。

### runtime.json 输入模式与求解进展日志

- 默认输入模式：`input_mode = static`。
- `static` 模式默认输入源：
	- `data/latest_small_tasks_pool.json`
	- `data/latest_windows.json`
- `simulation` 模式为可选调试输入：用于快速构造新快照与回归实验，不是默认生产输入路径。
- `solver_progress_enable`：默认 `true`，开启后记录求解过程进展日志。
- `solver_progress_every_n_solutions`：每找到 `N` 个中间解写 1 条进展记录；若 `N` 取值很大，则通常只在收尾阶段（FEASIBLE/OPTIMAL）保底写入。

运行后 `output/solver_progress.jsonl` 的每一行是一个 JSON 对象，字段契约如下：

- `solution_index`：当前中间解序号（从 1 开始累加）
- `objective`：该次中间解目标值
- `wall_time`：求解器墙钟时间（秒）
- `best_bound`：当前最优界

### 兼容迁移说明（旧键 -> 新键）

- 当检测到旧键且对应新键未配置时，加载/校验阶段会 `warnings.warn(...)`，并执行可预测映射：
	- `sequence_count_* + sequence_task_*` -> `structured_task_ratio`
	- `predecessor_probability`（或 `dag_chains_per_sequence_*`）-> `dependency_density`
	- `window_share_task_min/max` -> `window_reuse_target`
- 若已显式配置新键，则新键优先，不依赖旧键。
- `free_task_ratio` 作为兼容字段会由 `1 - structured_task_ratio` 自动回写，不建议继续作为主控参数。

## 任务时间语义（重构后）

- 窗口任务：`payload_type_requirements` 非空时，任务必须绑定 `visibility_window`。
- 自由任务：`payload_type_requirements` 为空且 `visibility_window = null`，表示不受可见窗口约束。
- 共享窗口：多个窗口任务可复用同一个 `visibility_window.window_id`，用于表达同一观测/通信可见时段下的任务簇。
- 求解时间域规则：
	- 有 `visibility_window`：任务开始/结束必须落在窗口 `[start, end]`。
	- 无 `visibility_window`：任务默认时间域为 `[0, time_horizon]`。
