# 隐式姿态约束与日志增强设计（定稿）

## 1. 背景与目标

本次设计用于增强静态基线任务规划的一期能力，聚焦以下三点：

1. 不再依赖输入 JSON 中的 `_att` 任务，由调度流程自动处理姿态切换。
2. 解决 CP-SAT 改变任务先后关系后，姿态切换时序随之变化的问题。
3. 增强日志可观测性，输出启发式阶段的初始化解与最终解（非摘要），并支持阶段化日志频率配置。

## 2. 定稿决策

### 2.1 总体建模策略

1. 启发式阶段：采用隐式姿态约束（不显式插入 ATT 任务参与求解）。
2. CP-SAT 阶段：采用隐式姿态约束（不在求解中动态增删任务变量）。
3. 输出阶段：按最终序列“物化”ATT 记录，形成紧邻绑定、可解释的最终计划输出。

### 2.2 初始姿态配置

1. `initial_attitude_angle_deg` 放在 `runtime.json`。
2. 无姿态任务执行后不改变当前姿态状态。

### 2.3 优化目标优先级

1. 第一优先级：关键任务完成率。
2. 第二优先级：总收益最大化。

## 3. 约束设计

### 3.1 启发式阶段（隐式姿态状态机）

在构造序列时维护状态：`current_time` 与 `current_attitude`。

当尝试安排候选任务 `j` 时：

1. 计算姿态差角：

   $$
   \Delta\theta = |\theta_{current} - \theta_j|
   $$

2. 计算转姿时间：

   $$
   t_{att} = \lceil \Delta\theta / \omega \rceil + buffer
   $$

3. 可行性检查：任务最早开始时间需满足时间窗、依赖、资源与转姿时间。
4. 若可行则放入并更新 `current_attitude = \theta_j`（任务有姿态目标时）。

### 3.2 CP-SAT 阶段（隐式转姿约束）

对于可能相邻的任务对，使用先后关系变量驱动转姿约束，而不是显式 ATT 任务变量。

当 `i` 在 `j` 前时，约束为：

$$
start_j \ge end_i + t_{att}(i,j)
$$

反向顺序时自动切换到对应约束。

首任务还需满足由 `initial_attitude_angle_deg` 推导的初始转姿约束。

该方式可自然适应 CP-SAT 对任务先后关系的调整，不需要在单次求解中动态增删变量。

### 3.3 输出后物化 ATT

在生成最终排程文件时，按最终顺序回放并插入 ATT 记录：

1. ATT 与对应业务任务紧邻绑定。
2. ATT 时长由最终相邻姿态差与配置参数实时计算。
3. ATT 用于输出解释与审计，不作为 CP-SAT 内部求解变量。

## 4. 日志设计

### 4.1 日志频率参数

拆分为两个参数：

1. `heuristic_log_every_n`
2. `cpsat_log_every_n`

### 4.2 必须输出的非摘要记录

启发式阶段新增两类全量记录（非摘要）：

1. `heuristic_initial_solution`
2. `heuristic_final_solution`

两类记录均包含任务序列、开始结束时间、关键任务统计、收益统计与姿态链信息。

### 4.3 过程摘要与终态

1. 启发式按 `heuristic_log_every_n` 输出中间摘要。
2. CP-SAT 按 `cpsat_log_every_n` 输出中间摘要。
3. 终态输出 `terminal` 汇总。
4. 统一写入 `solver_progress.jsonl`，通过 `event_type` 区分事件。

## 5. 组件边界与改动范围

预计涉及模块：

1. `src/scheduler/heuristic_scheduler.py`：隐式姿态状态机与日志增强。
2. `src/scheduler/cpsat_improver.py`：隐式转姿约束与过程日志频率。
3. `src/scheduler/pipeline.py`：阶段日志汇总与终态输出。
4. `src/scheduler/result_writer.py`：输出后 ATT 物化。
5. `src/scheduler/config.py`：新增日志频率与姿态参数读取校验。
6. `tests/`：新增/调整姿态约束与日志回归测试。

## 6. 失败与降级策略

1. 若任务不含姿态目标，则按“姿态不变”处理。
2. 若姿态参数缺失，配置校验阶段直接报错并中止。
3. 若输出物化 ATT 与内部时序冲突，视为程序错误并记录 `error` 级日志。

## 7. 验收标准

1. 不依赖输入 ATT 任务也可生成可行排程。
2. CP-SAT 调整任务顺序后，转姿约束仍保持可行性。
3. 输出包含紧邻 ATT 记录，且与最终顺序一致。
4. `solver_progress.jsonl` 包含：
   - 启发式初始化解（全量）
   - 启发式最终解（全量）
   - 两阶段中间摘要
   - 终态摘要

## 8. 非目标

1. 本次不实现多轮重建迭代优化（先保证单轮隐式建模稳定落地）。
2. 本次不扩展到动态重规划闭环。
