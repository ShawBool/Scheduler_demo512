# 任务模型重构计划（并发度与姿态角）

## 目标

1. 将 `Task.container_slots` 重构为 `Task.concurrency_cores`（并发核心占用）。
2. 将 `Task.attitude_mode` 重构为 `Task.attitude_angle_deg`（姿态角，单位度）。
3. 删除通信窗口与危险组合规则在当前阶段的建模与求解流程耦合（保留重规划模块独立存在）。
4. 保证相关联代码同步修改：模型、仿真、规划器、流水线、配置、测试、运行说明。

## 实施步骤

### 步骤1：模型与测试先行（TDD）

- 先改测试：
  - `tests/test_models.py`
  - `tests/test_simulation.py`
  - `tests/test_planner_constraints.py`
  - `tests/test_logging_utils.py`
  - `tests/test_pipeline.py`
- 先运行并确认失败（RED）。
- 再最小实现模型改动（GREEN）。

### 步骤2：核心调度逻辑改造

- `src/scheduler/models.py`
  - 新字段：`concurrency_cores`、`attitude_angle_deg`
  - `ScheduleItem` 改为记录姿态角
  - `ResourceSnapshot` 同步使用并发核心与姿态角
- `src/scheduler/planner.py`
  - 并发约束从 `container_capacity` 改为 `max_concurrency_cores`
  - 姿态切换时间从“固定时间”改为“角度差 × 系数”
  - 删除通信窗口与危险组合规则求解逻辑
- `src/scheduler/simulation.py`
  - 随机生成姿态角
  - 生成并发核心占用
  - 去除 `comm_kind`

### 步骤3：配置与流水线联动

- `config/planner_config.json`
  - `container_capacity -> max_concurrency_cores`
  - `attitude_switch_time -> attitude_time_per_degree`
  - 移除 `link_windows`、`danger_rules` 配置段
- `src/scheduler/config.py` 校验规则同步更新。
- `src/scheduler/pipeline.py` 日志指标去除通信/危险规则依赖。

### 步骤4：回归验证与提交

- 运行定向测试 + 全量测试 + 冒烟运行。
- 按里程碑提交：
  - 里程碑1：重构计划与配置约束调整
  - 里程碑2：模型与规划核心代码重构
  - 里程碑3：测试与运行产物验证

## 验收标准

1. `pytest` 全通过。
2. `main.py --seed 42` 可运行并产出调度结果。
3. 代码中不再依赖 `container_slots`、`attitude_mode`、`comm_kind`、`link_windows`、`danger_rules`。
4. 并发约束逻辑满足“插入任务后总并发不超过阈值”。
5. 姿态切换时间由角度差计算得到。
